# Resume & Cover-Letter Build Rules — <Developer Name>

## Resume rules
- Length: 1–2 pages when printed.
- Order of sections: Header → Summary → Skills → Experience → Education → Certifications.
- Tailoring: reorder and rewrite Experience bullets to surface the most JD-relevant
  work first. Mirror the JD's terminology where it truthfully applies.
- Keep it ATS-friendly: real text (no text-in-images), standard section headings,
  no multi-column layouts that break parsers.
- Always quantify impact when the profile provides numbers; never fabricate numbers.
- Tone: concise, results-oriented, third-person-implied (no "I").

## Cover-letter rules
- Length: 3–4 short paragraphs, max ~250 words.
- Open by naming the specific role and company.
- Middle: connect 2–3 of this developer's real achievements to the JD's top needs.
- Close with a clear, confident call to action.
- Tone: warm but professional.

## Styling / HTML
- Single-column, system font stack, generous whitespace, subtle accent color.
- Inline CSS only (must open standalone and print cleanly to PDF).
- (Optional) If a template.html exists in this folder, match its structure and styling.
