**Starter resume/cover‑letter rules – please review and edit as needed**

### Length & Overall Structure
- Keep the resume to **2 pages** (or ≤ 1 page for a targeted role) using a clean, single‑column layout.
- Order sections as: **Header → Summary → Experience → Skills → Education → Honors/Awards → Languages → Contact**.
- For cover letters, limit to **3‑4 short paragraphs** (≈ 150‑200 words).

### Tone & Content
- Use a **professional, results‑focused tone**; start bullet points with strong action verbs (e.g., *Led, Designed, Implemented*).
- Quantify impact wherever possible (e.g., “Reduced release cycle by 30%”). If exact numbers aren’t in the source, omit them – **don’t invent data**.
- Highlight **domain expertise** (Video Conferencing, Financial Services, Banking) and **key technologies** (Java, Spring MVC, Ext‑JS, Maven, Jenkins, JUnit, Selenium).

### ATS‑Friendliness
- **Plain text headings** (e.g., `## Experience`) and simple bullet points; avoid tables, graphics, or columns that can confuse parsers.
- Use **standard job titles** (e.g., “Senior Software Engineer”, “Software Engineer”) and **common keywords** from the target job description.
- Save the final file as **PDF/A** or a **Word document** with embedded fonts; retain searchable text.

### Styling Guidelines
- **Headers**: Bold company names, italicize dates/location.
- **Bullet points**: Keep to one‑line or two‑line statements; avoid long paragraphs.
- **Fonts**: Use a legible sans‑serif (e.g., Calibri 11 pt or Arial 10 pt).
- **Margins**: 0.75–1 in all sides; ensure sufficient white space.
- **Contact section**: Include only verified links (LinkedIn, GitHub if available) – omit empty fields.

### Customization Tips
- For each application, **tailor the Summary** to echo the most relevant experience for the role.
- Re‑order bullet points under each role to prioritize the achievements most aligned with the job posting.
- Add a **“Projects”** or **“Relevant Coursework”** subsection only if it showcases additional skills not covered in professional experience.

--- 

*These rules are a starting point. Adjust wording, ordering, and emphasis to match the specific role and company culture you’re applying to.*
